# regression_homework
Geron’s overall approach to predicting housing prices starts with understanding and preparing the data before doing any modeling. Instead of jumping straight into building a model, he first explores the dataset to get a sense of what each variable represents and how the data is distributed. He looks for patterns, checks for missing values, and identifies which features might be useful for prediction. This step is important because real-world data is often messy, and cleaning it properly makes a big difference in the final results.

Next, he prepares the data so it can be used effectively. This includes handling missing values, converting categories into a format the model can understand, and making sure numerical values are on comparable scales. He also creates meaningful groupings, such as income categories, to better capture patterns in the data. At this stage, the goal is to turn raw, inconsistent data into a clean and structured version that a model can learn from.

After preparing the data, he splits it into two parts: one for training the model and one for testing it. The training portion is used to teach the model how different features relate to housing prices, while the test portion is kept separate to evaluate how well the model performs on new, unseen data. He is careful to make sure this split is fair and representative, so the model is not biased by uneven data.

Finally, he builds and evaluates models, comparing different approaches to see which performs best. He measures how accurate the predictions are and adjusts the model to improve performance. Throughout the process, the focus is on avoiding overfitting—making sure the model doesn’t just memorize the training data but can generalize well to new data. Overall, the workflow emphasizes careful preparation, fair evaluation, and iterative improvement to arrive at a reliable prediction of housing prices.
